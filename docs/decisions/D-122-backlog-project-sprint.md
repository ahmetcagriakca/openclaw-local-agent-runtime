# D-122: Backlog-to-Project-to-Sprint Contract

**Phase:** 7 (Sprint 31)
**Status:** Frozen
**Date:** 2026-03-28

---

## Decision

GitHub Issues are the canonical source for backlog items. BACKLOG.md is a generated report. Backlog issues and sprint issues are separate entities with explicit linkage.

## Authority Model

| Source | Authority | Role |
|--------|-----------|------|
| GitHub Issues (label: `backlog`) | **Canonical** | Durable problem/feature inventory |
| BACKLOG.md | **Generated** | Read-only report, regenerated from GitHub |
| plan.yaml | **Sprint authority** | Frozen at sprint planning, automation index |
| Project V2 | **Display** | Reflects state, not a planning source |
| Milestone | **Sprint container** | Groups sprint issues |

## Identity Model

- **Backlog issue** = durable problem/feature (title: `[B-NNN] Description`)
- **Sprint issue** = execution slice (title: `[SN-N.M] Description`)
- Relationship: **1 backlog issue → 0..N sprint tasks across sprints**
- Sprint task body contains: `Relates to #<backlog_issue_number>`
- Backlog issue receives comment when sprint task is created

## Issue Labels

| Label | Purpose |
|-------|---------|
| `backlog` | Backlog item (durable) |
| `priority:P1` | High priority |
| `priority:P2` | Medium priority |
| `priority:P3` | Low priority |
| `sprint` | Sprint task (execution) |
| `security` / `product` / `operations` / `devex` | Domain labels |

## Project V2 Fields

| Field | Type | Values |
|-------|------|--------|
| Status | Single Select | Backlog, Planned, In Progress, Review, Done, Blocked |
| Priority | Single Select | P1, P2, P3 |
| Type | Single Select | Backlog, SprintTask |

## Project V2 Views

| View | Filter | Grouping |
|------|--------|----------|
| Backlog | label:backlog, Status != Done | Group by Priority |
| Active Sprint | current sprint milestone | Group by Status |

## Sprint Planning Flow

```
1. Operator reviews Backlog view in Project V2
2. Selects items for next sprint
3. Creates plan.yaml with backlog_ref for each task
4. issue-from-plan.yml creates sprint issues with backlog links
5. Sprint milestone groups execution issues
```

## Closure Sync

- Sprint task closed → update Project Status
- All sprint tasks for a backlog item done + acceptance met → close backlog issue
- Backlog issue close → Project Status = Done
- Reopen: only for regression, false closure, or incomplete acceptance
- New scope → new backlog issue (never reopen for expansion)

## BACKLOG.md Generation

- Generated by `tools/generate-backlog.py`
- Header: "Auto-generated from GitHub issues. Do not edit directly."
- Reflects current GitHub issue state
- Run on demand or as part of sprint closure
